# Exercice 12 - Récap final : créer un mini-site vitrine

> À faire dans le PDF : à la fin du cours, après le responsive design.

## Objectif

Réutiliser toutes les notions du cours dans un mini-site complet.

## Énoncé

Crée un mini-site vitrine pour un projet fictif ou réel. Le sujet est libre, mais il doit être assez riche pour intégrer du contenu, une navigation, des images, des cartes, un formulaire, un tableau et une version responsive.

Avant de coder, définis clairement le sujet, le public visé et l'objectif de la page. Ensuite, construis d'abord le HTML, puis ajoute progressivement le CSS.

## Idées de sujets

- café ou restaurant ;
- association ;
- club de sport ;
- portfolio personnel ;
- événement ;
- petite entreprise ;
- atelier créatif ;
- service local.

## Fichiers attendus

Le rendu doit contenir :

- un fichier `index.html` ;
- un fichier `style.css` ;
- un dossier `images` ;
- au moins deux images utilisées dans la page.

## Contraintes HTML

- La page doit avoir une structure HTML complète.
- La langue et l'encodage doivent être définis.
- Le viewport doit être présent.
- Le titre de l'onglet doit être personnalisé.
- La page doit contenir `header`, `nav`, `main` et `footer`.
- La page doit contenir au moins trois sections.
- La page doit contenir au moins deux articles.
- La page doit contenir un aside.
- La page doit contenir plusieurs niveaux de titres.

## Contraintes de contenu

- La page doit contenir au moins quatre paragraphes.
- Une liste à puces doit être présente.
- Une liste ordonnée doit être présente.
- Au moins deux liens internes avec ancres doivent fonctionner.
- Au moins un lien externe doit être présent.
- Au moins deux images doivent être présentes.
- Les images doivent avoir un texte alternatif utile.
- Au moins une image doit avoir une légende.

## Contraintes CSS

- Le CSS doit être séparé du HTML.
- Une police générale doit être définie.
- Les couleurs doivent être cohérentes.
- Les marges et paddings doivent être travaillés.
- Au moins une classe réutilisable doit être utilisée.
- Au moins un identifiant doit être ciblé.
- Des bordures, ombres ou coins arrondis doivent être utilisés.
- Un effet au survol doit être présent.
- Un effet de focus doit être présent.
- Une media query doit adapter la page aux petits écrans.

## Contraintes formulaire

- Le site doit contenir un formulaire de contact.
- Le formulaire doit demander au minimum un nom, un email, un sujet et un message.
- Les labels doivent être liés aux champs.
- Au moins deux champs doivent être obligatoires.
- Le bouton d'envoi doit être présent.

## Contraintes tableau

- Le site doit contenir un tableau utile au sujet choisi.
- Le tableau doit avoir un titre.
- Le tableau doit avoir des en-têtes.
- Le tableau doit contenir au moins trois lignes de données.
- Le tableau doit être stylisé en CSS.

## Contraintes responsive

- Les images ne doivent pas déborder.
- Le menu doit rester utilisable.
- Le formulaire doit rester lisible.
- Le tableau doit rester consultable.
- La page doit être agréable à lire sur petit écran.

## Résultat attendu

Le mini-site doit ressembler à une page vitrine complète et cohérente. On doit comprendre le sujet, naviguer dans la page, lire les informations, remplir le formulaire et consulter le tableau.

## Checklist finale

- Le site s'ouvre dans un navigateur.
- Le HTML et le CSS sont séparés.
- Le fichier CSS est lié correctement.
- Les accents s'affichent correctement.
- Les images s'affichent.
- Les images ont un `alt`.
- Le menu fonctionne.
- Les ancres fonctionnent.
- Le formulaire est utilisable.
- Le tableau est lisible.
- La page est responsive.
- Je suis capable d'expliquer mon code.
